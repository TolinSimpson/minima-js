Getting Started with MinimaJS

Learn how to build your first MinimaJS application in minutes

1Installation

Choose your preferred installation method:

Option A: CDN (Recommended for beginners)

The fastest way to get started. No downloads or setup required.

<!DOCTYPE html> <html> <head> <title>My MinimaJS App</title> </head> <body> <div id="app"></div> <!-- MinimaJS Core --> <script src="https://cdn.jsdelivr.net/npm/minimajs/src/minima.js"></script> <!-- Development Tools (optional, for development only) --> <script src="https://cdn.jsdelivr.net/npm/minimajs/src/minima.dev.js"></script> <script> // Your app code goes here </script> </body> </html>

Option B: Local Development

Download and run locally for full development experience with hot reload.

# Clone the repository git clone https://github.com/minimajs/minimajs.git cd minimajs # Start the development server npm run dev # Open your browser to http://localhost:8080
💡 Tip: Use the CDN method for learning and prototyping. Switch to local development when you need hot reload and development tools.

2Your First Component

Let's create a simple counter component to understand the basics:

// Define a component with enhanced features const Counter = MinimaJS.defineComponent(function(props) { // Enhanced reactive state with computed properties this.state = MinimaJS.createState({ count: props.initialCount || 0, name: props.name || 'Counter' }, { // Computed property - automatically updates when count changes doubleCount: function(state) { return state.count * 2; }, isEven: function(state) { return state.count % 2 === 0; } }); // Component methods (available in templates) this.increment = () => { this.state.count++; }; this.decrement = () => { this.state.count--; }; this.reset = () => { this.state.count = 0; }; // Lifecycle hooks this.onInit = function() { console.log(`${this.state.name} component initialized!`); }; this.onDestroy = function() { console.log(`${this.state.name} component destroyed!`); }; // Enhanced template with interpolation and event binding return ` <div style="text-align: center; padding: 20px; border: 1px solid #ddd; border-radius: 8px;"> <h2>{{state.name}}: {{state.count}}</h2> <p>Double: {{state.doubleCount}} | {{state.isEven ? 'Even' : 'Odd'}}</p> <button @click="increment" style="margin: 5px; padding: 10px 15px;">+</button> <button @click="decrement" style="margin: 5px; padding: 10px 15px;">-</button> <button @click="reset" style="margin: 5px; padding: 10px 15px;">Reset</button> </div> `; }); // Render the component MinimaJS.render(Counter, document.getElementById('app'));

Understanding the Component

  • defineComponent: Creates a new component with lifecycle management
  • this.state: Reactive state that triggers re-renders when changed
  • Event handlers: Regular JavaScript functions that modify state
  • Template: HTML template string with JavaScript interpolation

3Props and Communication

Components receive data through props and can communicate with parent components:

// Child component that receives props const Greeting = MinimaJS.defineComponent(function(props) { return ` <div style="padding: 10px; background: #f0f0f0; margin: 10px 0; border-radius: 4px;"> <h3>Hello, ${props.name}!</h3> <p>You are ${props.age} years old.</p> ${props.isAdmin ? '<p>👑 Admin privileges enabled</p>' : ''} </div> `; }); // Parent component that passes props const UserProfile = MinimaJS.defineComponent(function(props) { this.state = MinimaJS.createState({ user: { name: 'John Doe', age: 25, isAdmin: false } }); const toggleAdmin = () => { this.state.user = { ...this.state.user, isAdmin: !this.state.user.isAdmin }; }; // Render child component with props const greetingHTML = Greeting({ name: this.state.user.name, age: this.state.user.age, isAdmin: this.state.user.isAdmin })._template; return ` <div> ${greetingHTML} <button onclick="toggleAdmin()"> ${this.state.user.isAdmin ? 'Remove Admin' : 'Make Admin'} </button> </div> `; });

4Global State Management

Share data across components using global stores:

// Create a global store const appStore = MinimaJS.store('app', { theme: 'light', user: null, notifications: [] }); // Component that uses global state const ThemeToggle = MinimaJS.defineComponent(function(props) { const toggleTheme = () => { appStore.theme = appStore.theme === 'light' ? 'dark' : 'light'; }; return ` <div style="padding: 15px; background: ${appStore.theme === 'dark' ? '#333' : '#fff'}; color: ${appStore.theme === 'dark' ? '#fff' : '#333'};"> <h3>Current theme: ${appStore.theme}</h3> <button onclick="toggleTheme()">Switch to ${appStore.theme === 'light' ? 'Dark' : 'Light'} Mode</button> </div> `; }); // Another component that reacts to the same store const ThemeDisplay = MinimaJS.defineComponent(function(props) { // Subscribe to store changes this.onInit = function() { MinimaJS.onGlobalStateChange('app', 'theme', () => { // Component will re-render automatically console.log('Theme changed to:', appStore.theme); }); }; return ` <p>The current theme is: <strong>${appStore.theme}</strong></p> `; });

5Lifecycle Hooks

Components have lifecycle hooks for initialization and updates:

const LifecycleDemo = MinimaJS.defineComponent(function(props) { this.state = MinimaJS.createState({ mounted: false, updateCount: 0, data: null }); // Called once when component is first rendered this.onInit = function() { console.log('Component initialized!'); this.state.mounted = true; // Simulate data fetching setTimeout(() => { this.state.data = 'Data loaded from API'; }, 1000); }; // Called every time the component re-renders this.onUpdate = function() { console.log('Component updated!'); this.state.updateCount++; }; const triggerUpdate = () => { this.state.mounted = !this.state.mounted; }; return ` <div style="padding: 15px; border: 1px solid #ddd;"> <h3>Lifecycle Demo</h3> <p>Mounted: ${this.state.mounted}</p> <p>Updates: ${this.state.updateCount}</p> <p>Data: ${this.state.data || 'Loading...'}</p> <button onclick="triggerUpdate()">Trigger Update</button> </div> `; });
